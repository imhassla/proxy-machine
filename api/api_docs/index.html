Proxy-Machine API Documentation

Harvest → validate (anonymity-classified, geo/ASN-enriched) → store → serve. The API is read-only; the rotating relay (:3333) and SOCKS5 listener (:1080) egress through the validated pool. All endpoints are same-origin.

GET /proxy/{type}

Returns validated proxies of type ∈ http | https | socks4 | socks5, fastest first. An empty match is 200 with an empty body. Query params:

  • time — max response time in seconds, e.g. ?time=1.5
  • minutes — max age since last check (default 30; 0 disables)
  • format — json (default) | text | csv | curl (paste-ready curl -x lines) | proxychains
  • anon — anonymity tier: elite | anonymous | unknown (transparent proxies are never stored)
  • country — 2-letter code, e.g. ?country=US (needs geo enrichment)
  • asn — substring of "AS<n> org", e.g. ?asn=cloudflare
  • pick=1 — return ONE round-robin proxy; session=ID pins a session to one proxy (sliding TTL); add rotate=1 to force a fresh pick

Examples: /proxy/http?minutes=0&format=text · /proxy/socks5?country=US&anon=elite · /proxy/http?format=curl

Integration & observability

  • GET /proxy.pac — browser proxy auto-config (relay + fastest http fallbacks)
  • GET /stats — JSON: proxy counts per type + relay counters
  • GET /insights — JSON: anonymity tiers, latency histogram, top countries & networks (ASN), fastest proxies
  • GET /upstreams — JSON: live relay upstream health (ewma latency, consecutive fails, circuit state)
  • GET /metrics — Prometheus text format
  • GET /ready — 200 once ≥1 validated upstream (readiness) · GET /health — liveness
  • GET /dashboard — live retro dashboard
